You type “compare refugee numbers in Egypt and Jordan” like any other message, and back comes a table with the real figure, its year and its source. Built for research, journalism and aid work. No code, no spreadsheets.

01

Why does AI get refugee numbers wrong?

Try it now: ask any AI chatbot how many refugees your country hosts. You will usually get a stale number from its training data, a loose estimate, or a polite apology.

Even an assistant that queries the UN database by itself falls into its trap: it codes countries with a scheme that contradicts the global standard in 99 of 232 countries — Egypt's code there is ARE, which everywhere else means the United Arab Emirates — and a wrong query returns no warning, just an empty list, so the assistant answers with confidence: “no data.”

This server defused all of those traps once and for all — so your assistant queries the official database the right way, every time.

Without the server

you › How many registered refugees are in Egypt?

assistant › “Egypt hosts hundreds of thousands of refugees — likely around 300,000 as of my last data (2023), though the numbers may have changed…”

A guess from memory: approximate, stale, unsourced.

With humanitarian-mcp

you › The same question.

assistant › “220,033 registered refugees in Egypt at the end of 2025 — source: UNHCR, Refugee Data Finder.”

A live query: exact, dated, sourced.

03

Who benefits? And how?

Anyone who needs a trustworthy figure on displacement and asylum — without learning technical tools or spending the day inside spreadsheets:

The academic researcher

Forced migration studies · political economy · conflict research

Asks: “Give me the series of people displaced from Sudan 2010–2025 as CSV, with anomalous years flagged”

Gets: clean data ready for R, Stata or Python, with unified ISO codes and a documented source to cite — in seconds instead of a day of data cleaning.

The data journalist

Investigations · crisis coverage · fact-checking claims

Asks: “Is Uganda really among the top five refugee-hosting countries this year?”

Gets: an up-to-date ranking from official figures (yes — third-largest host in 2025) + a publication-ready chart + the attribution line — before the deadline.

The aid organization

Grant writing · donor briefings · program planning

Asks: “Prepare a donor briefing on the situation in Chad, including the age structure of new arrivals”

Gets: a complete Markdown report through the built-in donor_briefing prompt — headline figures, a ten-year trend, women and children shares — ready to edit instead of a week of manual compilation.

The student and the teacher

Theses · presentations · data-science classes

Asks: “Explain the trend in Syrian asylum over the last ten years with a simple chart”

Gets: a plain-language explanation + a Mermaid chart that pastes straight into the deck, and a source that can be verified.

Ready in two minutes.

Fastest path: the “No install” tab below — a hosted URL we run for you; paste it and start in seconds. Or install it on your machine in three steps that need no programmer, and your questions and data stay entirely on your device.

  1. 1 · Install Node.js if it is not on your machine — a free, ordinary click-through install.
  2. 2 · The package is published on npm — no cloning, no building. Your app's config (tabs below) simply points at npx humanitarian-mcp. For a quick try in the terminal:
npx humanitarian-mcp --version

3 · Pick your app from the tabs, add the config snippet, restart it — then ask in your own words.

A ready hosted instance — no terminal, no install. Copy the URL and paste it into your app:

https://humanitarian-mcp.zad.tools/mcp
  • In Claude (web or app): Settings → Connectors → Add custom connector → paste the URL.
  • In Claude Code: claude mcp add --transport http humanitarian https://humanitarian-mcp.zad.tools/mcp

The hosted instance relays your questions through our server (no accounts, no tracking, and the data is public anyway). For full privacy or offline work, use the local install above.

Add this to claude_desktop_config.json and restart Claude — then ask in your own words. 

{
  "mcpServers": {
    "humanitarian": {
      "command": "npx",
      "args": ["-y", "humanitarian-mcp"]
    }
  }
}

One command registers the server for all sessions:

claude mcp add humanitarian -- npx -y humanitarian-mcp

Then try:

  • “Write a full humanitarian report on Sudan”
  • “Export Jordan's asylum decisions 2018–2025 as a CSV file”
  • “Which countries saw anomalous displacement surges since 2015?”

A stateless HTTP endpoint — every request stands alone, works behind any load balancer. 

npm run dashboard   # POST /mcp on port 8642

curl -s -X POST http://localhost:8642/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{
        "name":"top_host_countries","arguments":{"limit":5}}}'

No MCP client? The built-in dashboard shows the providers, the catalogue and live logs — and can run any tool from the browser. 

npm run dashboard
# → http://localhost:8642

Optional settings: HMCP_CACHE=sqlite keeps data between sessions · HMCP_OFFLINE=1 serves from the cache only — useful in the field where there is no connection.

Three workshops — one per profession

Complete paths from question to deliverable: for the researcher, the journalist and the organization — with real numbers, not mock examples. And remember: the dark lines below are written by the assistant itself behind the scenes; you only type the question.

1 · For the researcher: studying an event's impact

The question: what did the April 2023 war do to the numbers displaced from Sudan?

The trend tool gives you the full series, the linear fit, and flags the break years statistically — then exports a clean CSV that drops straight into your interrupted time-series model. 

# in the conversation
trend_analysis({ country: "Sudan", role: "origin", year_from: 2010 })
→ anomalies: [{ year: 2023, changePct: +78.8, zScore: 2.6 }]

export_data({ dataset: "population", country: "Sudan",
              role: "origin", format: "csv", year_from: 2010 })

# then in Python
import pandas as pd
df = pd.read_csv("sudan.csv")     # country_code, year, refugees…
df["post_war"] = (df.year >= 2023).astype(int)
# interrupted time-series as usual

2 · For the journalist: checking a claim before deadline

The question: an official says Uganda is among the top five refugee-hosting countries — publish the claim as is?

The ranking tool returns the list updated with official UNHCR figures, and the chart tool hands over a publish-ready SVG with the attribution line — before the issue closes. 

top_host_countries({ metric: "refugees", year: 2025 })
→ Uganda: #3 globally (2025) — claim verified ✓

generate_chart({ countries: ["Uganda"], format: "svg" })
# → SVG ready to publish, source line included

# fact-box footer, ready to paste:
# "Source: UNHCR Refugee Data Finder, 2025"

3 · For the organization: a donor briefing from the built-in prompt

The question: a donor briefing on the situation in Chad is due next week, not next month.

The built-in donor_briefing prompt appears with one click in the prompt list and steers the assistant through the right tool sequence — including the shares that protection and education programs are built on and grants are written with. 

# built-in prompt — one click in Claude Desktop
donor_briefing({ country: "Chad", audience: "institutional donors" })
→ runs country_profile · trend_analysis · demographics
→ delivers a Markdown briefing, ready to edit

# the number programs are built on, directly:
demographics({ country: "Egypt" })
→ total: 1,098,306 · female: 54.2% · children (0–17): 39.1%

Are we the first? Honestly: no — and that is our advantage.

Earlier tools already connect AI assistants to refugee data (most notably rvibek/mcp_unhcr) and there is a strong tool for the HDX platform (dividor/hdx-mcp). But they pass the raw data through as is — their user must know the codes, compute the rates and draw the charts. This project was built to be the understanding layer, not just a pipe:

08

Questions before you try it

What does “AI assistant” mean here? +

AI chat programs: you type a question, they answer. The one this server is most used with is the free Claude Desktop app for the computer — and it works with any program that supports the open MCP standard, a list that keeps growing.

Is it free? +

Yes. The project is open source under the MIT license, and UNHCR data is public and needs no API keys — the server itself costs nothing. The MCP client you choose (Claude Desktop, for instance) has its own plans, and the server works with their free tiers.

Do I need to know how to code? +

No. Installation is one terminal command and one config file — the “Get running” section above walks through it step by step. After that everything happens in your own words, inside the chat.

How current are the figures? Are they official? +

They come straight from UNHCR's official API, with a local cache that refreshes automatically. Figures are end-year stocks, and the latest years may be preliminary mid-year estimates — every output carries its reference year and source so you can verify it yourself.

Where do my questions and data go? +

Nowhere. The server runs locally on your machine, read-only, and talks only to the UNHCR API — no tracking, no third parties. And with HMCP_OFFLINE=1 it serves from the cache with no connection at all.

Can I ask in Arabic or other languages? +

Ask in any language your assistant understands — Arabic included; even Arabic country names («مصر», «السودان») resolve correctly. The tools themselves are language-neutral and return structured data, and the assistant writes the answer in the language of your question.

Are the forecasts official UN planning figures? +

No. They are simple linear projections, clearly labeled as such — good for sensing direction, not for official planning, and they must not be attributed to UNHCR.

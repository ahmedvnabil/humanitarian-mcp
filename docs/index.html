The API is booby-trapped.
We defused it once.

UNHCR's API uses its own country codes that contradict ISO for 99 of 232 countries. Egypt is ARE — which is the UAE's ISO code. Guess wrong and you don't get an error. You get an empty array, and your assistant confidently reports nothing.

raw REST

GET /population/v1/population/?coa=EGY

{ "items": [] }   // silent. wrong code.

humanitarian-mcp

refugee_population({ country: "Egypt" })

{ country: "Egypt", country_code: "EGY",
  year: 2025, refugees: 220,033, ... }
Normalized. ISO3 + consistent fields across every future provider.
Read-only. Every tool is annotated; nothing writes upstream.
Polite. Rate-limited, retried, cached with ETag revalidation.
Cited. UNHCR attribution travels with every payload.

Use it in two minutes.

Add to claude_desktop_config.json, restart Claude, then just ask questions.

{
  "mcpServers": {
    "humanitarian": {
      "command": "node",
      "args": ["/path/to/humanitarian-mcp/dist/index.js"]
    }
  }
}

One command registers the server for every session.

claude mcp add humanitarian -- node /path/to/humanitarian-mcp/dist/index.js

Then, inside Claude Code:

› Generate a donor briefing about Sudan.
› Which countries saw anomalous displacement spikes since 2015?
› Export Jordan's asylum decisions 2018–2025 as CSV.

Stateless streamable HTTP — every request self-contained, trivially load-balanced.

npm run dashboard   # serves POST /mcp on :8642

curl -s -X POST http://localhost:8642/mcp \
  -H 'content-type: application/json' \
  -H 'accept: application/json, text/event-stream' \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/call","params":{
        "name":"top_host_countries","arguments":{"limit":5}}}'

No MCP client handy? The built-in dashboard shows providers, health, the full catalogue, live logs — and runs any tool from the browser.

npm run dashboard
# → http://localhost:8642

Environment knobs (all optional): HMCP_CACHE=sqlite persists data between runs · HMCP_OFFLINE=1 serves cache only · full list in .env.example ↗
